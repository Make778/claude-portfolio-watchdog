# `portfolio/` — твои CSV-выгрузки из IBKR

Положи сюда CSV-файл с твоими позициями из Interactive Brokers.
Скрипт автоматически найдёт **последний по времени** файл и распарсит его.

> ⚠️ **Безопасность.** Система **только читает** содержимое CSV — никаких
> подключений к брокерскому API, никаких автоматических сделок.
> Эти файлы остаются в твоём приватном репозитории.

---

## Поддерживаемые форматы

Парсер автоопределяет формат по содержимому и пробует все варианты:

### 1. **Activity Statement** (стандартный экспорт IBKR)

Многосекционный CSV, нужная секция — `Positions`. Выгружается через:
**IBKR Client Portal → Performance & Reports → Statements →
Activity → Format: CSV**.

Пример (упрощённо):
```
Statement,Header,Field Name,Field Value
Statement,Data,BrokerName,Interactive Brokers
...
Positions,Header,Symbol,Quantity,Mult,Cost Price,Cost Basis,Close Price,Value,...
Positions,Data,AAPL,100,1,150.25,15025,180.50,18050,...
Positions,Data,TSLA,50,1,250.00,12500,200.00,10000,...
```

### 2. **Flex Query** (продвинутый экспорт IBKR)

Обычный однотаблично-структурированный CSV. Выгружается через:
**Reports → Flex Queries → Custom Flex → CSV**. Полезно, если хочешь
сразу указать, какие колонки нужны.

### 3. **Portfolio Analyst** (упрощённая выгрузка)

Простая таблица позиций без вспомогательных секций.

### 4. **Простой CSV** (если ты сделал файл руками)

Минимальный формат:
```csv
ticker,quantity,price
AAPL,100,150.25
TSLA,50,250.00
NVDA,30,500.00
```

Колонки:
- `ticker` — биржевой тикер;
- `quantity` — количество акций;
- `price` (опционально) — средняя цена покупки в USD.

---

## Если CSV нет

Система переключится на список `tickers:` из `config.yaml` и будет
следить за ними без P&L (только сигналы по тикерам).

---

## Что игнорируется

- Закрытые позиции (`quantity == 0`).
- Forex (`asset_type == "FX"`).
- Опционы (`asset_type == "OPT"`) — логируются, но не уходят в основную аналитику.
- Дубликаты тикеров автоматически суммируются.

---

## Юридическое предупреждение

Claude Portfolio Watchdog не подключается к брокерскому API, не совершает
сделок, не передаёт данные третьим лицам. Все файлы в этой папке
остаются в твоём приватном GitHub-репозитории.
