# Расходы на Claude API

Полная разбивка ожидаемых расходов при модели
`claude-sonnet-4-20250514` и портфеле из **10–15 позиций**.

---

## Краткий вывод

**~$1.50 / месяц** при дефолтных настройках. **GitHub Actions
бесплатно** (Free план = 2000 минут CI/CD на приватные репо в месяц,
наши workflow уложатся в 200–400).

## Что делает кеш

`data/news_cache.json` хранит оценки Claude по заголовкам на 7 дней.
Без него на каждую новость по каждому тикеру тратился бы Claude-вызов
(~30 в день × 10 тикеров = 300/день).

Этот файл **коммитится в git** и тем самым переживает рестарты Actions.

## Как уменьшить расход в 2–3 раза

1. **Поднять порог важности новостей**
   `thresholds.news_importance_min: 8` (по умолчанию 7) — отсечёт
   "полу-релевантные" новости и снизит количество Claude-вызовов на скоринг.

2. **Отключить макро и волатильность**
   ```yaml
   monitoring:
     macro: false
     volatility: false
   ```
   Эти мониторы шлют меньше алертов, но всё равно дёргают Claude для
   generate_insights в макро-сигналах.

3. **Сократить tickers**
   Чем меньше бумаг, тем меньше новостей → меньше скоринга.

4. **Не запускать monitor чаще нужного**
   Дефолтный cron `0 13-20 * * 1-5` — это 8 запусков в день. Если тебе
   достаточно 2 — поменяй на `0 13,17 * * 1-5`.

## Лимиты GitHub Actions Free

- **2000 минут / месяц** на приватных репо (на публичных — бесплатно
  без ограничений).
- Один запуск monitor занимает **30–90 секунд**.
- Daily summary — ~20 секунд.
- Weekly PDF — ~2 минуты.

**Месячный итог:** ~8 monitor × 22 дня = ~176 запусков по 1 мин = ~180
минут. + daily (~22 × 0.5 мин = 11) + weekly (~4 × 2 = 8). **Итого
~200 минут** — 10% от free-лимита.
